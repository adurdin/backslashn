In TextMate, this is done with a custom Python script that examines the selection, and outputs a snippet. The snippet is dynamically generated so that it can contain the necessary line breaks and indentation already. It is then the snippet that handles the mirroring of the start tag to the end tag, with a regex substitution applied to omit the whitespace and any attributes after it.

This command has been invaluable to me when editing HTML (which I do a lot), but I have been unable to figure out how to write a similar command within Sublime Text 2. Multiple selections would seem to be obvious for mirroring the start tag content to the end tag, but without the crucial omission of attributes. So for now I have the inconvenient situation of using Sublime Text 2 for editing most files, but opening up TextMate whenever I’m working with HTML.
